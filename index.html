hexo-prism-plugin

post @ 2021-09-04

首次配置 Hexo 并使用 hexo-prism-plugin 代码高亮插件

  1. 安装 hexo-prism-plugin 插件, 并且需要同时安装 hexo-inject 插件, 命令如下:

    $ npm install hexo-prism-plugin --save
$ npm install hexo-inject --save

  2. 修改 _config.yml 配置文件, 禁用 Hexo 的 highlight 如下（将 highlight 的 enable 修改为 false）:

    highlight:
enable: false

  3. 禁用 Hexo 的 prismjs 配置, 如下（直接注释）:

    #prismjs:
#  enable: true
#  preprocess: true
#  line_number: true
#  tab_replace: ''

  4. 在配置文件 _config.yml 的最后一行后面追加以下内容:

    prism_plugin:
mode: 'preprocess'    #realtime/preprocess
theme: 'gruvbox-dark' #这是prism插件的代码高亮主题，根据个人喜好修改
enable: true
preprocess: true
line_number: true
tab_replace: ''

GDB调试命令

post @ 2021-09-04

GDB调试基础命令

命令 描述
list 列出源代码，接着上次的位置往下列，每次列10行
list 行号 列出从几行开始的源代码
start 开始执行程序
help 查看帮助
info 查看变量的值
info locals 查看局部变量的值
backtrace 查看各级函数及调用参数
frame 栈帧号 选择栈帧
next 执行下一行语句
step 执行下一行语句，如果有函数调用则进入到函数中
finish 让程序一直运行到从当前函数返回为止
set var 变量=值 修改变量的值, 修改变量的值除了用set命令之外也可以用print命令，因为print命令后面跟的是表达式，而我们知道赋值和函数调用也都是表达式，所以也可以用print命令修改变量的值或者调用函数
display 变量名 跟踪查看某个变量，每次停下来都显示它的值
undisplay 跟踪显示号 取消跟踪显示变量值
breakpoint 设置断点
continue 连续运行程序而非单步运行
info breakpoints 查看已经设置的断点
delete breakpoint 断点编号 删除某个断点
disable breakpoint 断点编号 禁用某个断点
run 从头开始连续运行程序
break 函数名 在某个函数开头设置断点
break … if … 设置条件断点
print *数组名称@长度 查看数组各项的值，例如：print *array@5
watch 设置观察点
info watchpoints 查看当前设置了哪些观察点
x 从某个位置开始打印存储单元的内容，全部当成字节来看，而不区分哪个字节属于哪个变量。例如：x/7b，7b是打印格式，b表示每个字节一组，7表示打印7组
